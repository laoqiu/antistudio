# 连线样式和UI顺序修复

## 修复概述

1. **已存在的连线样式同步** - 初始边也使用自定义样式
2. **输出格式位置调整** - 移动到输入参数之后

## 修复 1: 初始连线样式同步

### 问题
初始示例中的连线还在使用旧的 `smoothstep` 类型，没有应用新的自定义样式（贝塞尔曲线 + 删除按钮）。

### 修复前
```typescript
const initialEdges: Edge[] = [
  {
    id: 'e1',
    source: 'start',
    target: 'llm-1',
    type: 'smoothstep',  // ❌ 旧类型
    animated: true,
    style: { stroke: '#71717a' }
  },
  // ...
];
```

### 修复后
```typescript
const initialEdges: Edge[] = [
  {
    id: 'e1',
    source: 'start',
    target: 'llm-1',
    type: 'custom',  // ✅ 自定义类型
    markerEnd: { type: MarkerType.ArrowClosed, color: '#71717a' },
    style: { stroke: '#71717a', strokeWidth: 2 }
  },
  // ...
];
```

### 效果
- ✅ 所有连线（包括初始的）都显示为贝塞尔曲线
- ✅ 所有连线都有删除按钮
- ✅ 统一的视觉风格

## 修复 2: 输出格式位置调整

### 问题
输出格式选择在输入参数之前，逻辑顺序不够直观。用户希望先配置输入，再选择输出格式。

### 修复前的顺序
```
1. 输出格式选择
2. ─────────────────
3. 输入参数配置
4. ─────────────────
5. 输出变量配置
```

### 修复后的顺序
```
1. 输入参数配置
2. ─────────────────
3. 输出格式选择
4. ─────────────────
5. 输出变量配置
```

### 优点
- ✅ 更符合从上到下的配置流程
- ✅ 输入 → 格式 → 输出，逻辑更清晰
- ✅ 格式选择和输出配置在一起，关联性更强

## 修改的代码位置

### 文件
`frontend/components/workspace/WorkflowEditor.tsx`

- **行数**: ~1559-1572
- **改动**: 2 个边的类型从 `smoothstep` 改为 `custom`
- **新增**: `markerEnd` 和更新的 `style`

### 修改点 2: llm-text 配置
- **行数**: ~1051-1100
- **改动**: 将"输出格式选择"部分移动到"输入参数"之后
